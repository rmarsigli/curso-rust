# Fundamentos

## O que foi feito?

* Ownership básico
* Borrowing (`&`, `&mut`)
* `Option<T>` e `Result<T,E>`
* TODO CLI (Vec, push, remove)
* Error handling básico

## Resumo do Professor

Você começou do zero absoluto em Rust, sem conhecimento prévio da linguagem. Passou pela primeira experiência com ownership, borrowing e o sistema de tipos rigoroso do Rust — conceitos completamente diferentes de PHP/JavaScript. Construiu um TODO CLI funcional usando Vec, aprendeu a manipular dados mutáveis com `&mut`, e teve o primeiro contato com `Option<T>` e `Result<T,E>`, os substitutos do null/exception do Rust.

O mais importante desse dia não foi o código em si, mas o choque cultural: você percebeu que Rust não é "PHP rápido" ou "JavaScript seguro", é uma filosofia completamente diferente. O borrow checker te frustrou algumas vezes (normal), mas você não desistiu — continuou até o código compilar e funcionar. Isso demonstrou logo no primeiro dia que você tem a persistência necessária para aprender Rust.

Ao final, você tinha um CLI funcional que adiciona, lista e remove tasks de um vetor. Simples, mas com fundação sólida: entendeu que ownership move valores por padrão, que `&` empresta imutavelmente e `&mut` empresta mutavelmente, e que Rust força você a lidar com erros (Option/Result) em compile-time. Primeira aula completa, \~1 hora de estudo, código funcionando, base estabelecida.
